# Test Cases Repository

This repository contains test cases for multiple modules and features of various applications.  
The purpose of this repository is to organize manual test cases and automation test scenarios in a clear and accessible structure.

---

##  Folder Structure

Each folder represents a separate module or feature.

---

##  Included Modules

- **Pay Bill Module**
- **Photo Upload Module**
- **Transaction API Module**
- (Additional modules will be added as needed).

---

##  Test Case Types

This repository includes:

- **Manual Test Cases**
- **Functional Testing**
- **UI Testing**
- **API Testing**
- **Regression Test Cases**
- **Negative & Edge Case Scenarios**

---


##  Purpose

- To maintain a well-structured library of test cases  
- To document and organize test cases as part of skill development and best practices.  
- To demonstrate test design and documentation skills

###  Notes
